The First Line of Defense:
Input Validation

Dalam dunia software engineering, ada satu aturan emas: "Don't Trust User Input". Sebagian besar bug, crash, dan celah keamanan (hacking) terjadi karena programmer terlalu percaya bahwa user akan selalu mengetik hal yang benar.

üí° Fact: Input adalah pintu gerbang utama data masuk ke sistem. Jika pintu ini tidak dijaga, sampah dan virus bisa masuk dengan mudah.

// Vulnerable Code

# ‚ùå DANGEROUS!
# Apa yang terjadi jika user
# mengetik "sepuluh"?

>> CRASH: ValueError!

Logic Protection: Menjaga Aturan Bisnis üëÆ

Disinilah kita butuh Business Logic Validation.

Kasus: Stok Negatif

Toko online tidak boleh punya stok -5 barang.
Secara matematika: -5 adalah Integer (Valid secara Tipe).
Secara bisnis: Mustahil (Invalid secara Logika).

if stok < 0:
  print("Error: Stok tidak boleh minus!")

Kasus: Range Nilai

Nilai ujian maksimal 100.
Jika user input 999, itu Integer valid, tapi logic error.

  print("Error: Nilai max 100!")

The Shield Pattern: Guard Clauses üõ°Ô∏è

Daripada membuat if-else yang bertingkat-tingkat (seperti tangga), gunakan "Pola Perisai" (Guard Clauses) untuk menolak error di awal.

The Arrow (Miring)
def daftar(umur):
  if umur > 0:
    if umur >= 17:
      print("Boleh")
    else:
      print("Kecil")
  else:
    print("Error")
‚ùå Susah dibaca (Deep nesting).
The Shield (Datar)
def daftar(umur):
  # üõ°Ô∏è Guard 1: Cek Error dulu
  if umur <= 0: return "Error"

  # üõ°Ô∏è Guard 2: Cek Syarat
  if umur < 17: return "Kecil"

  # ‚úÖ Lolos semua perisai
  return "Boleh"
‚úÖ Rapi dan mudah dipahami.
